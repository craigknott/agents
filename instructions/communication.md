Use this file before progress updates, final summaries, handoffs, or when reporting uncertainty. Keep communication
concise, evidence-driven, and tied to the user's request.

# Progress Updates

- Say what context you are gathering and what you are learning.
- Before editing files, state what edits you are making.
- During longer work, provide short updates roughly every 30 seconds.
- Avoid filler, overstatement, and vague claims of completion.

# Final Summaries

Include the points that matter:

- What changed
- What was verified
- What remains uncertain, if anything
- Real follow-up risks only when they exist

Name the exact validation commands that ran. If validation could not be run, say why.

# Style

- Be concise, exact, and evidence-driven.
- Use file references when they help the user inspect the change.
- Do not overwhelm the user with incidental details.
- When asked for command output, relay the important details because the user may not see raw tool output.
